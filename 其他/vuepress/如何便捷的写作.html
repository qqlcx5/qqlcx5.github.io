# 如何便捷的写作

# 前言

这篇总结自己对markdown中图片有效的处理和部署等的一些经验之谈，为了更便捷的写作。

目前模式：

在MWeb上写作后，通过MwebGithub项目上传文章的图片，然后生成markdown文件到vuepress项目中，上传到github上，GitHub Actions 自动打包部署。最后通过bloghelper项目将文章发布到多端平台。

# 写作

当你在MWeb中写好一篇文章，准备发布的时候，里面的图片就是一个难以处理的麻烦。如果将文章在某个主流平台写的话，该平台基本会设置图片防盗链，这就意味着复制到其他平台，有得重新处理图片问题，这本身不是我们想看到的。所以我在找能支持Markdown语法的相关写作软件，经过尝试多款相关软件，最终MWeb更佳符合预期。

MWeb这个软件我觉得不错的功能，可以将本地图片上传服务器，然后将图片转换成服务器的地址。

然而这个功能并不是非常完善，不能将图片上传github，思前想后在百度搜了一下，找了GitHub相关的项目，通过MWeb自定义接口，在本地启动一个服务，上传到github服务器上。

# 用法

  • git clone https://github.com/qqlcx5/MWebPictures.git 克隆项目
  • npm install 安装依赖
  • 修改 config.json
  • node index.js 启动服务
  • 在MWeb中添加发布服务

# 先创建存放图床github仓库

仓库创建成功后，克隆我的仓库，然后将里面图片删除掉，上传到刚创建的仓库上。

git clone https://github.com/qqlcx5/figure-bed

图床里面有deploy.sh就是用来发布版本号的。

这个免费图床有个缺陷，当前分支大小超过 50M，jsDelivr 就无法获取文件的问题，尽量在仓库要满50M就发布一个版本。

然后回到MWeb项目中，配置config.json

# config.json 说明

{
	"repo": "qqlcx5/figure-bed", // github仓库地址
	"token": "b71420e517b40...", //个人访问令牌
	"port": 9000,
	"url": "/upload"
}

不懂了解token可以查看我之前发布教程。
